Descrição Resumida do Projeto

Ojectivo Geral

Desenvolvi um simulador de escalonamento de CPU em C++ que implementa 6 algoritmos de escalonamento (3 não preemptivos e 3 preemptivos), lendo processos de um ficheiro de texto e escrevendo os tempos de espera (Waiting Times) e a média num ficheiro de saída, tudo sem utilizar arrays.

Algoritmo Tipo Número
FCFS (First-Come, First-Served) Não Preemptivo 1
SJF (Shortest Job First) Não Preemptivo 2
SRTF (SJF Preemptivo) Preemptivo 3
Priority (Prioridade) Não Preemptivo 4
Priority Preemptivo Preemptivo 5
Round Robin (com quantum) Preemptivo 6

Estrutura do Programa

  1. Leitura de Argumentos
    • -t → quantum (1–255) para o Round Robin
    • -f → ficheiro de entrada com os processos
    • -o → ficheiro de saída para os resultados
  2. Ficheiro de Entreda
    • Formato: burstTime:arrivalTime:priority (um processo por linha)
    • Leitura dinâmica (número ilimitado de processos)
  3. Estruturas de Dados
    • struct Process → contém ID, burstTime, arrivalTime, priority, waitingTime, Remaing_time
    • struct node → nó da lista ligada para os processos
    • struct ResultsNode → nó da lista ligada para os resultados
  4. Operações com Listas Ligadas (sem arrays)
    • insertBack / insertBackRR → inserção no fim
    • insertBurstOrdered → inserção ordenada por burstTime
    • insertPriorityOrdered → inserção ordenada por prioridade
    • insertRemaingOrdered → inserção ordenada por remainingTime
    • removeHead → remoção do primeiro nó
    • copyList → cópia profunda da lista
    • freeProcess / freeResults → libertação de memória
  5. Algoritmos de Escalonamento
    • FCFS() → fila FIFO, não preemptivo
    • SJF() → fila ordenada por burstTime, não preemptivo
    • SJF_Preemptive() → fila ordenada por remainingTime, preemptivo (SRTF)
    • Priority() → fila ordenada por prioridade, não preemptivo
    • Priority_Preemptive() → fila ordenada por prioridade, preemptivo
    • RR() → fila FIFO com quantum, preemptivo
  6. Fila de Prontos
    • Lista ligada que contém os processos prontos para executar
    • Processos são movidos da lista notReady para a Ready quando arrivalTime <= tempo_atual
    • Cada algoritmo ordena a fila de forma diferente
  7. Cálculo do Waiting Time
    • Não preemptivos: waitingTime = tempo_atual - arrivalTime
    • Preemptivos: waitingTime = tempo_termino - arrivalTime - burstTime (calculado no término)
  8. Ficheiro de Saída
    • Formato: algoritmo:wt1:wt2:...:media (com duas casas decimais)
    • Exemplo: 1:0:4:8:10:13:7.00
    • Algoritmos não implementados escrevem 0:0:0:...:0.00

Desafios Superados

  • Sem arrays: todas as estruturas são listas ligadas com alocação dinâmica
  • Gestão de memória: new e delete para evitar memory leaks
  • Ordenação dinâmica: inserção ordenada em listas ligadas (bubble sort adaptado)
  • Preempção: controlo de processos interrompidos e reinserção na fila
  • Processos com chegadas diferentes: lista notReady para processos que ainda não chegaram
  • Cópia da lista original: cada algoritmo trabalha com a sua própria cópia

Tecnologias Utilizadas

  • Linguagem: C++ (sem usar arrays)
  • Compilador: g++ no Linux
  • Bibliotecas: iostream, fstream, string, iomanip, cstdlib

Exemplo de Execução

Execução:

./cmpe351 -t 2 -f input.txt -o output.txt

input.txt:

0:5:2

1:3:1

2:3:3

Output

1:0:4:6:3.33

2:0:6:3:3.00

3:5:2:0:2.33

4:4:0:7:3.67

5:3:0:6:3.00

6:3:3:4:3.33

Resumo Final

O projeto consistiu na implementação completa de um simulador de escalonamento de CPU em C++, utilizando listas ligadas como estrutura principal (sem arrays), com 6 algoritmos de escalonamento, leitura dinâmica de ficheiros, argumentos por linha de comando e geração de ficheiro de saída com os tempos de espera. O programa é robusto, trata memory leaks e suporta qualquer número de processos.
